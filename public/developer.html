Plugin Development Documentation

This documentation provides comprehensive information about developing plugins for KushAI.

Plugin Structure

Every KushAI plugin must include the following files:

  • manifest.json - Plugin metadata and configuration
  • plugin.py - Main plugin implementation
  • README.md - Documentation and usage instructions
  • config.json - Optional configuration file

Manifest File

The manifest.json file contains metadata about your plugin:

{
  "name": "My Plugin",
  "version": "1.0.0",
  "author": "Your Name",
  "description": "A brief description of what your plugin does",
  "license": "MIT"
}

Plugin Implementation

Your plugin must inherit from the PluginBase class and implement the required methods:

from plugin_base import PluginBase
from typing import Dict, Any, Generator

class MyPlugin(PluginBase):
    def get_tool_definition(self) -> Dict[str, Any]:
        return {
            "name": "my_plugin_tool",
            "description": "Description of what this tool does",
            "parameters": [
                {
                    "name": "param1",
                    "type": "string",
                    "description": "Description of param1",
                    "required": True
                }
            ]
        }

    def execute_tool(self, parameters: Dict[str, Any]) -> Generator[str, None, None]:
        # Implementation of your tool
        yield "Result from my plugin tool"

Plugin API

Plugins have access to the following KushAI APIs:

  • self.assistant - Access to the main assistant instance
  • self.plugin_dir - Path to the plugin's directory
  • self.config - Plugin configuration data
  • self.get_config_value(key, default) - Get a configuration value
  • self.set_config_value(key, value) - Set a configuration value
  • self.save_config() - Save configuration to file

Developer Tools

We provide several tools to help you develop and test your plugins.

Plugin Validator

Our plugin validator checks your plugin for common issues and ensures it meets our requirements.

# Run the validator on your plugin
python validate_plugin.py /path/to/your/plugin

Plugin Tester

Test your plugin in a simulated KushAI environment before publishing.

# Test your plugin
python test_plugin.py /path/to/your/plugin

Debugging Tools

Enable debug mode to get detailed logs and error information:

# Enable debug mode
export KUSHAI_DEBUG=1
python kush_ai.py

Performance Profiler

Analyze your plugin's performance and identify bottlenecks:

# Profile your plugin
python profile_plugin.py /path/to/your/plugin

Publish Your Plugin

Share your plugin with the KushAI community by publishing it to the Plugin Marketplace.

Submission Requirements

  • Plugin must pass our validation checks
  • Plugin must include comprehensive documentation
  • Plugin must not contain malicious code
  • Plugin must follow our coding standards
  • Plugin must have a valid license

Submission Process

  1. Fork the KushAI-Plugins repository
  2. Create a new branch for your plugin
  3. Add your plugin to the plugins directory
  4. Update the plugins/index.json file with your plugin's metadata
  5. Submit a pull request with your plugin

Review Process

Our team will review your plugin for:

  • Functionality and correctness
  • Security and privacy compliance
  • Documentation quality
  • Code quality and best practices

After Publication

Once your plugin is published:

  • It will appear in the Plugin Marketplace
  • Users can install and rate your plugin
  • You'll receive analytics about plugin usage
  • You can update your plugin by submitting new pull requests
